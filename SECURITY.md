## Security Considerations

### Client-Side App
Lê is a **static progressive web app** with no backend. All data (progress, SRS cards, preferences) is stored in the browser's `localStorage`. This means:

- **No server-side vulnerabilities**: there's no database to breach or API to exploit.
- **User data is local-only**: we don't collect, transmit, or store user data on any server.
- **Supply chain**: the app's security depends on npm dependencies and the build pipeline.

### Known Risks & Mitigations

#### 1. Cross-Site Scripting (XSS)
**Risk**: User-generated content (e.g. pasted Portuguese text in the Reader) could contain malicious scripts.

**Mitigation**:
- All user input is rendered via React, which escapes HTML by default.
- The mini-markdown renderer in `src/lib/miniMarkdown.tsx` does NOT use `dangerouslySetInnerHTML`.
- No `eval()` or `Function()` constructors anywhere in the codebase.

**Action**: If you find a way to inject script tags via the Reader input or any other field, please report it.

#### 2. Dependency Vulnerabilities
**Risk**: npm packages may contain known vulnerabilities.

**Mitigation**:
- We run `npm audit` regularly and update dependencies.
- GitHub Dependabot is enabled (if repository is on GitHub) to auto-create PRs for security patches.

**Action**: Run `npm audit` before every release. Fix high/critical vulnerabilities immediately.

#### 3. Content Security Policy (CSP)
**Risk**: Without a CSP header, the app is more vulnerable to XSS.

**Mitigation**:
- The app is served from Cloudflare Pages, which allows custom headers via `_headers` file.
- TODO: Add a strict CSP in `public/_headers`:
  ```
  /*
    Content-Security-Policy: default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data:; font-src 'self'; connect-src 'self'; media-src 'self'; object-src 'none'; frame-ancestors 'none'; base-uri 'self'; form-action 'self'
  ```
  (Adjust as needed for the service worker and any CDN resources.)

**Action**: If deploying to production, add a CSP header.

#### 4. Service Worker Cache Poisoning
**Risk**: A malicious actor could poison the service worker cache.

**Mitigation**:
- The service worker is generated by `vite-plugin-pwa` in `generateSW` mode, which precaches only build-time assets (hashed filenames).
- We don't cache user-uploaded files or external resources.
- The SW script itself is served with `Cache-Control: no-cache` so updates are fetched on every page load.

**Action**: If you find a way to inject malicious content into the SW cache, please report it.

#### 5. localStorage Data Integrity
**Risk**: An attacker with local access could modify `localStorage` to corrupt SRS state or inject fake progress.

**Mitigation**:
- This is a client-side-only app with no server-side verification, so local tampering is possible but only affects the attacker's own device.
- There's no incentive to cheat (no leaderboard, no certificates, no accounts).
- If we add cloud sync in the future, we'll implement server-side validation.

**Action**: Not a security concern for the current architecture. Monitor if cloud sync is added.

#### 6. Third-Party Audio/Speech APIs
**Risk**: The app uses the Web Speech API (`SpeechSynthesis`) for text-to-speech, which is a browser API with no external network calls by default. Phoneme audio files are bundled or sourced from Wikimedia Commons.

**Mitigation**:
- We don't send user text to external TTS services.
- Audio files are served from the same origin (or a trusted CDN).

**Action**: If we add external TTS (e.g. Google Cloud TTS), we must disclose this in the privacy policy and get user consent.

## Privacy

Lê does not collect, transmit, or store user data on any server. All data (progress, SRS cards, recent texts, preferences) is stored in the browser's `localStorage` and never leaves the device.

### Analytics
The app **does not use analytics** (no Google Analytics, no tracking pixels). If we add analytics in the future, we will:
- Use privacy-respecting analytics (e.g. Plausible, Fathom)
- Disclose it in the README and add an opt-out

### Third-Party Services
- **Cloudflare Pages**: The app is hosted on Cloudflare Pages. Cloudflare may log IP addresses and user-agent strings for DDoS protection and analytics (standard web hosting behavior). See [Cloudflare's Privacy Policy](https://www.cloudflare.com/privacypolicy/).
- **No other third-party services** are used.
